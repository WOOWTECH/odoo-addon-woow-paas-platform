Complete AI Automation Solution

AI-Powered Automation

Create automation rules and scheduled actions that execute AI actions with custom prompts. AI analyzes data and makes intelligent decisions automatically.

Smart Record Processing

AI automatically processes records when automation rules trigger or scheduled actions run. AI can create, update, classify, and analyze records based on your prompts.

Context-Aware Actions

AI receives full record context automatically. All triggered records are tagged in prompts using $model/id syntax for intelligent processing.

Non-Interactive Execution

AI runs in automation mode, executing actions directly without asking for confirmation. Perfect for automated workflows and background processing.

Flexible Triggers

Use Automation Rules for event-based triggers (on create, on update) or Scheduled Actions for time-based automation (daily, weekly, monthly). AI actions execute automatically.

Enterprise Security

AI respects user permissions and data access controls. All automation actions follow the same security policies as manual operations.

How It Works - 3 Simple Steps

1

Create Automation Rule or Scheduled Action

Go to Settings Technical Automation. Create Automation Rules for event-based triggers or Scheduled Actions for time-based tasks. Select your trigger or schedule.

2

Configure AI Action

Set Action Type to 'AI Action'. Select your AI Assistant and write a prompt describing what AI should do when the rule triggers.

3

Automatic Execution

When trigger conditions are met or schedule arrives, AI automatically processes records according to your prompt. AI can create, update, analyze, and make decisions.

Technical Specifications

System Requirements

  • Odoo 18.0
  • ai_base_gt module
  • base_automation module
  • PostgreSQL 13+ with pgvector extension
  • Minimum 8GB RAM (16GB recommended)
  • Internet connection for AI APIs

AI Features & Tools

  • AI integration with automation rules & scheduled actions
  • Automatic record context attachment
  • Record tagging in prompts ($model/id syntax)
  • Non-interactive AI execution mode
  • Full permission and security validation
  • Multi-record batch processing
