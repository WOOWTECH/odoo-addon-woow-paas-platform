# Task: Verify n8n Helm chart and API Key support

## Description
驗證 n8n-mcp 整合的三個關鍵技術假設，結果決定後續 tasks 的實作方式。這是 critical path task，應最先執行。

## Acceptance Criteria
- [ ] 確認 n8n Helm chart 是否支援 `extraContainers` / `sidecars` values key
- [ ] 確認 n8n 是否支援 `N8N_API_KEY` 環境變數預設 API Key
- [ ] 確認 n8n-mcp Docker image 的健康檢查端點（`/health` 或其他）
- [ ] 確認 n8n-mcp HTTP 模式支援的 transport（SSE / Streamable HTTP）
- [ ] 記錄驗證結果和備選方案到 epic 文件

## Technical Details
- **n8n Helm chart**: `oci://8gears.container-registry.com/library/n8n`
  - 下載 chart 並檢查 `values.yaml` 和 deployment template
  - 如不支援 extraContainers，研究替代方案（PaaS Operator K8s patch、自定義 chart）
- **n8n API Key**: 查閱 n8n 文件 https://docs.n8n.io
  - 檢查環境變數 `N8N_API_KEY` 或類似配置
  - 如不支援，研究 n8n REST API 自動建立 API Key
- **n8n-mcp image**: `ghcr.io/czlonkowski/n8n-mcp:latest`
  - 本地 docker run 測試 HTTP 模式
  - 確認 health endpoint 和 MCP transport endpoint
- 驗證結果記錄到 `.claude/epics/n8n-mcp-integration/research-notes.md`
