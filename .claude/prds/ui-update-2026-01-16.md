# PRD: UI 更新 2026-01-16

## 執行摘要

將 WoowTech PaaS 平台前端 UI 從設計版本 `2026-01-14` 更新至 `2026-01-16`。此次更新簡化導航結構、增強頁首的使用者資訊顯示，並重新設計儀表板內容以提升易用性。

**設計參考**: `resource/stitch_paas_web_app_shell_global_navigation_2026-01-16/`

## 問題陳述

目前的 UI 實作基於 2026-01-14 設計規格，存在以下問題：
- 複雜的導航結構，包含 5 個選單項目（含「Deployments」）
- 頁首資訊不足，缺乏使用者資訊和通知功能
- 儀表板聚焦於系統指標，而非以使用者為中心的概覽

新的 2026-01-16 設計透過更簡潔、更以使用者為中心的介面解決這些問題。

## 使用者故事

### US-1: 簡化導航
**作為**平台使用者
**我希望**側邊欄選項更簡潔
**以便**我能更有效率地導航，不會感到困惑

**驗收條件：**
- 側邊欄顯示 4 個主要項目：Dashboard、Workspaces、Billing、Settings
- 移除「Deployments」選單項目
- 側邊欄底部顯示「Log Out」連結
- 保留「Help & Support」連結

### US-2: 增強頁首
**作為**已登入的使用者
**我希望**在頁首看到我的帳戶資訊和通知
**以便**我能快速存取點數餘額、通知和個人資料

**驗收條件：**
- Credits 徽章顯示目前餘額，附帶加值按鈕
- 通知鈴鐺圖示顯示未讀數量徽章
- 顯示使用者名稱和方案類型
- 顯示使用者頭像（縮寫或照片）

### US-3: 改進儀表板概覽
**作為**工作區管理員
**我希望**在儀表板看到團隊和帳務資訊
**以便**我能一目了然地監控資源使用狀況

**驗收條件：**
- 成員概覽卡片顯示：總成員數、活躍使用者、待處理邀請
- 帳務概覽卡片顯示：本月用量、剩餘點數、下次帳單日期
- 工作區概覽卡片顯示：雲端服務數量、安全通道數、工作區數量

### US-4: 簡化 Logo
**作為**使用者
**我希望** Logo 呈現更簡潔
**以便**介面感覺更現代、更清爽

**驗收條件：**
- Logo 僅顯示「WoowTech」文字
- 移除 Logo 區域的「CONSOLE」副標題

## 需求規格

### 功能需求

#### FR-1: 側邊欄導航更新
| 項目 | 舊版 (01-14) | 新版 (01-16) |
|------|-------------|-------------|
| 選單項目 | Dashboard, Workspaces, Deployments, Billing, Settings | Dashboard, Workspaces, Billing, Settings |
| 底部區塊 | Help & Support, 使用者資訊 | Help & Support, Log Out |
| 使用者資訊 | 顯示在側邊欄 | 移至頁首 |

#### FR-2: 頁首元件更新
| 元素 | 舊版 (01-14) | 新版 (01-16) |
|---------|-------------|-------------|
| 左側 | 麵包屑 | 麵包屑（不變） |
| 中間 | - | - |
| 右側 | 僅 Balance 按鈕 | Credits 徽章 + 通知 + 使用者資訊 + 頭像 |

#### FR-3: 儀表板頁面更新
| 區塊 | 舊版 (01-14) | 新版 (01-16) |
|---------|-------------|-------------|
| 摘要卡片 | Active Instances, Secure Tunnels, System Health | Members Overview, Billing Overview, Workspace Overview |
| 近期活動 | 是（不變） | 是（不變） |
| 快速存取 | 是（不變） | 是（不變） |

#### FR-4: Logo 更新
- 移除「CONSOLE」副標題文字
- 保留 WoowTech 品牌圖示和主文字

### 非功能需求

#### NFR-1: 效能
- 頁首使用者資料不增加額外網路請求（使用 session 資料）
- 側邊欄過渡動畫保持流暢（< 100ms）

#### NFR-2: 無障礙性
- 所有互動元素保持鍵盤可導航
- 新頁首元素更新 ARIA 標籤
- 維持色彩對比度

#### NFR-3: 響應式設計
- 頁首元素在行動裝置上適當堆疊
- 側邊欄收合行為不變

## 限制與假設

### 限制
- 必須在現有 OWL 框架內運作
- 不能修改 Odoo 核心資源
- 設計基於 Stitch 匯出（HTML/CSS 參考）

### 假設
- 使用者 session 資料包含點數餘額
- 通知數量 API 端點存在或將使用模擬資料
- 設計標記（顏色、間距）保持一致
